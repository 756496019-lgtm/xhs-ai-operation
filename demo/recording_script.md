# Demo 录屏分镜｜5-7 分钟端到端

> 目标：5-7 分钟内呈现从 CSV 导入到下期选题反哺的完整闭环，每一步都"真的在跑"。

## 录制环境

- 屏幕分辨率：1920×1080，16:9
- 录屏工具：OBS（推荐）/ Bandicam / 系统自带
- 帧率：30 fps（够用）
- 音频：可选旁白（个人讲解节奏更好）；不录旁白则字幕条说明
- 字幕：底部黑底白字字幕条，4-5 秒一句

## 整体节奏

| 时间 | 内容 | 类型 |
|---|---|---|
| 0:00-0:30 | 开场 + 真实战绩 | 截图 + 字幕 |
| 0:30-1:30 | 一句话出片演示（creator/xhs_console_agent） | 实跑 |
| 1:30-2:30 | 多源监控 + 双路径改写 + fact_check（crawler/content-monitor） | 实跑 |
| 2:30-3:30 | 对标 taptap 账号监控（benchmark） | 实跑 |
| 3:30-5:00 | 数据回收闭环（analytics）：导入 → 分析 → 卡片报告 → 下期选题 | 实跑 |
| 5:00-6:00 | 闭环演示：选题 prompt 复制回 creator 起新生产 | 实跑 |
| 6:00-7:00 | 收尾：回到粉丝曲线，强调"AI 用越少越好" | 截图 + 字幕 |

## 详细分镜

### 0:00-0:30｜开场

```
[镜头：近 30 天粉丝增长曲线截图（创作中心可导这一段）+ 旁白补全程]

字幕：
"近 30 天 +XXX 粉，前 3 个月已累计 2700+，全程 4 个月 0 → 2978"
"AI 辅助下沉淀的工作流，不是为了展示搭的样板间"

[切到：仓库结构截图（VS Code 资源管理器侧边栏 6 个子目录）]

字幕：
"6 个环节，6 个子目录，1 个闭环"
```

### 0:30-1:30｜一句话出片（creator）

```
[镜头：WorkBuddy 终端窗口，cd 到 creator/xhs_console_agent/]

操作员输入：
> 帮我做一条本周折扣盘点的小红书图文，标题方向是「Steam 夏促值得买的5款新游」

[等 WorkBuddy 自己读 SKILL.md，自己跑爬虫，自己生成文案，自己出图]

[加速 4-8 倍播放]

字幕：
"WorkBuddy 自动读 skill 文档，跑爬虫，写文案，出图"
"运营全程不敲一行命令"

[镜头：output/ 目录里的成品（cover.jpg + script.txt + 6 张图文卡）]
```

### 1:30-2:30｜多源监控 + 改写 + 事实核查（crawler）

```
[镜头：浏览器打开 http://localhost:5050（content-monitor）]

字幕：
"日常素材池：14 个 scrapers"
"Reddit / 微博超话 / B站 / TapTap / 触乐 / 17173 / Epic / Steam ..."

[勾选 3 条新闻，点"文字改写"]

字幕：
"双路径改写：文字 / 视频脚本"

[等改写完，点"事实核查"按钮]

[弹窗：黄色"有疑问"，列出 1-2 条无源断言]

字幕：
"W17 踩过编数字的坑，现在每篇先过 fact_check"
"无源断言全部标黄"
```

### 2:30-3:30｜对标 taptap 监控（benchmark）

```
[镜头：终端 cd benchmark/]

操作员输入：
> python monitor_taptap.py

[实时输出：拉到 N 条笔记，写入 data/taptap_YYYYMMDD.json]

[VS Code 打开 taptap_YYYYMMDD.json，展示 3-5 条笔记]

字幕：
"用 Agent-Reach OpenCLI 复用浏览器登录态"
"跨周追踪对方的选题节奏"
"反哺到 analytics 做'对方在打但我没打'分析"
```

### 3:30-5:00｜数据回收闭环（analytics 核心展示）

```
[镜头：终端 cd analytics/]

操作员输入：
> python cli.py --import examples/sample_export.csv --analyze --suggest --render

[终端逐步输出：
  - 解析 CSV：N 条笔记 / M 条快照入库
  - 分析中：互动率 / 时段 / 标题模式
  - AI 复盘中：调 qwen-max
  - 渲染卡片中：双风格 HTML 输出
  - 选题反哺中：next_week_prompt_W{N}.md 已生成]

字幕：
"创作中心 CSV → SQLite → AI 复盘 → 双风格卡片报告"

[浏览器打开 data/reports/W{N}_dashboard.html]

[展示双风格切换：暗黑 / 奶油]

字幕：
"复盘结果直接长成下期能发的小红书卡片样式"

[点下载按钮，导出 JPG（在浏览器侧边栏看到下载列表）]

字幕：
"一键导 JPG，复盘和发布共用同一套视觉规范"
```

### 5:00-6:00｜闭环演示（最关键的 1 分钟）

```
[镜头：VS Code 打开 next_week_prompt_W{N}.md]

[滚动到末尾，高亮"WorkBuddy 触发话术"那一段]

字幕：
"复盘 → 下期选题 prompt"

[复制最后一段]

[切到 creator/xhs_console_agent/ 终端窗口]

[粘贴话术，回车]

[WorkBuddy 开始自动出片]

字幕：
"复制回 creator，起一轮新生产 → 闭环成立"
"这就是赛题要求的'基于阅读/互动数据优化下一轮选题'"
```


## 录屏前准备清单

- [ ] 准备 sample_export.csv（脱敏，10-20 行典型数据）
- [ ] 提前跑通 `python cli.py --import ... --analyze --suggest --render` 一次，确保不会中途报错
- [ ] benchmark 的 Agent-Reach 提前装好，`opencli doctor` 通过
- [ ] content-monitor 的 Flask 提前 `python app.py` 起好
- [ ] 浏览器收藏夹放 3 个常用页面：`localhost:5050`、`data/reports/W*.html`
- [ ] 终端字号调大（让观看者看得清）
- [ ] 关闭微信 / 钉钉 / 邮件提醒
- [ ] 桌面整洁，关闭无关窗口
- [ ] 准备好近 30 天粉丝增长曲线截图（创作中心可导，再用旁白补全程 4 个月数据）

## 节奏控制技巧

- 长任务（如 WorkBuddy 出片）用录屏后处理 4-8 倍速，但保留前后 1 秒原速作为锚点
- 每个段落结束都切回 README 或仓库结构图 1 秒，给观看者一个"回到地图"的感觉
- 字幕条不要太密——4-5 秒一句，每句 ≤ 15 字
- 不录旁白时，背景音乐用 youtube 免版权（节奏轻快，不抢戏）

## 时长冗余

目标 6-7 分钟。如果实测 < 5 分钟，加 1-2 分钟在 3:30-5:00 段（数据回收）的细节展示——这是核心创新点。
如果 > 7 分钟，砍 0:30-1:30 的一句话出片演示（这部分价值最低，因为它和很多参赛者类似）。
